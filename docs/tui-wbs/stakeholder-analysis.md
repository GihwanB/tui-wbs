# 이해관계자 분석: TUI WBS

본 문서는 TUI WBS 프로젝트가 다양한 이해관계자의 요구사항을 충족하는지 확인하기 위해 작성되었습니다.

## 1. 소프트웨어 개발자 (주요 사용자)
**페르소나**: CLI 도구를 선호하고, 키보드 중심의 워크플로우를 가지며, Git을 사용하여 버전을 관리합니다.
**요구사항**:
- **속도**: 빠른 실행과 터미널 워크플로우와의 매끄러운 통합이 필요합니다.
- **키보드 효율성**: vim 스타일의 키바인딩이 필수적입니다.
- **Git 친화적**: 파일 포맷은 diff 확인이 쉬워야 합니다 (Markdown은 이에 매우 적합합니다).
- **컨텍스트 전환 최소화**: 작업 상태를 업데이트하기 위해 터미널을 떠나고 싶어하지 않습니다.
**우려사항**:
- **복잡성**: 설치나 사용법이 너무 복잡하면 다시 일반 텍스트 편집기로 돌아갈 것입니다.
- **파일 손상**: "Round-trip" 무결성이 매우 중요합니다. 자동화 도구가 자신이 작성한 주석이나 포맷을 망가뜨리면 안 됩니다.

## 2. 프로젝트 관리자 (PM - 보조 사용자)
**페르소나**: 일정을 관리하고 작업을 할당하며 진행 상황을 추적합니다. 기술적 배경이 있을 수도 있고 없을 수도 있습니다.
**요구사항**:
- **전체 개요**: 전체 그림(트리 뷰)을 쉽게 볼 수 있어야 합니다.
- **상태 추적**: 완료된 작업, 진행 중인 작업, 지연된 작업을 빠르게 파악해야 합니다.
- **메타데이터 관리**: 문법을 신경 쓰지 않고도 담당자, 날짜, 우선순위를 쉽게 업데이트할 수 있어야 합니다.
**우려사항**:
- **진입 장벽**: 엄격한 문법 규칙을 외워야 한다면 채택률이 낮을 것입니다. TUI가 이 간극을 메워줘야 합니다.
- **이동성**: 이 도구가 없는 사람도 파일을 볼 수 있어야 합니다 (네, Markdown이므로 가능합니다).

## 3. 협업자 / 리뷰어 (단순 열람자)
**페르소나**: WBS를 읽어야 하지만 TUI 도구를 사용하지 않을 수도 있습니다 (예: GitHub/GitLab에서 조회).
**요구사항**:
- **가독성**: 파일은 표준 Markdown으로서 읽을 수 있어야 합니다. 메타데이터를 HTML 주석으로 처리하는 것은 렌더링 된 뷰를 깔끔하게 유지하면서 데이터를 구조화하는 현명한 선택입니다.
- **접근성**: 표준 헤더(` #`)를 사용하여 어떤 Markdown 뷰어에서도 쉽게 탐색할 수 있어야 합니다.
**우려사항**:
- **숨겨진 정보**: 주석으로 처리된 중요 정보(상태, 담당자 등)가 렌더링 된 Markdown 뷰어에서는 보이지 않을 수 있습니다. *완화 방안: 핵심 정보는 텍스트에도 반영하거나, 메타데이터 전용임을 인지시켜야 합니다.*

## 4. DevOps / 시스템 관리자
**페르소나**: 환경 설정이나 파이프라인 구축을 담당합니다.
**요구사항**:
- **단순한 의존성**: Python 3.12+와 `textual` 정도면 합리적입니다.
- **설치 편의성**: `pip install` 또는 `mise` 통합이 필요합니다.
**우려사항**:
- **호환성**: Linux(서버), Mac(개발자), Windows에서 모두 작동하나요? Textual은 일반적으로 지원하지만 테스트가 필요합니다.

## 5. 프로덕트 오너 (PO - 전략적 관점)
**페르소나**: 비전과 가치를 정의합니다.
**분석**:
- **가치 제안**: 복잡한 GUI 프로젝트 관리 도구(Jira, Asana)와 원시 텍스트 파일 사이의 간극을 메웁니다.
- **시장 적합성**: 개발자 중심 팀에게 작지만 강력한 틈새 시장이 있습니다.
- **비용**: Python/Textual 생태계를 활용하여 유지보수 비용이 낮습니다.

---
## 이해관계자 분석을 통한 핵심 요구사항 요약
1.  **엄격한 Round-trip 보장**: 개발자의 신뢰를 얻기 위해 필수적입니다.
2.  **하이브리드 가독성**: TUI에서도, 원시/렌더링 된 Markdown에서도 모두 보기 좋아야 합니다.
3.  **낮은 마찰**: vim 키바인딩과 즉각적인 로딩 속도가 필요합니다.
4.  **표준 준수**: 다른 생태계 도구와의 호환성을 유지하기 위해 가능한 한 표준 Markdown 기능을 사용해야 합니다.
